Defined by the Parameters keyword written in a Markdown list item:<br /></p><p>+ Parameters<br /></p><p>Discussion of URI parameters in the scope of the parent section.<br /></p><p><br /></p><p>This section must be composed of nested list items only. This section must not contain any other elements. Each list item describes a single URI parameter. The nested list items subsections inherit from the Named section and are subject to additional formatting as follows:<br /></p><p>+ <parameter name>: `<example value>` (<type> | enum[<type>], required | optional) - <description><br /></p><p>    <additional description><br /></p><p>    + Default: `<default value>`<br /></p><p>    + Members<br /></p><p>        + `<enumeration value 1>` - <enumeration description 1><br /></p><p>        + `<enumeration value 2>` - <enumeration description 2><br /></p><p>        ...<br /></p><p>        + `<enumeration value N>` - <enumeration description N><br /></p><p>Where:<br /></p><p><br /></p><p><<b>parameter name</b>> is the parameter name as written in Resource Section's URI (e.g. "id").<br /></p><p><<b>description</b>> is any optional Markdown-formatted description of the parameter.<br /></p><p><<b>additional description</b>> is any additional optional Markdown-formatted description of the parameter.<br /></p><p><<b>default value</b>> is an optional default value of the parameter – a value that is used when no value is explicitly set (optional parameters only).<br /></p><p><<b>example value</b>> is an optional example value of the parameter (e.g. 1234).<br /></p><p><<b>type</b>> is the optional parameter type as expected by the API (e.g. "number", "string", "boolean"). "string" is the default.<br /></p><p>Members is the optional enumeration of possible values. <type> should be surrounded by enum[] if this is present. For example, if enumeration values are present for a parameter whose type is number, then enum[number] should be used instead of number to.<br /></p><p><<b>enumeration value n</b>> represents an element of enumeration type.<br /></p><p><<b>enumeration description n</b>> is the optional description of the corresponding enumeration element.<br /></p><p>required is the optional specifier of a required parameter (this is the default)<br /></p><p>optional is the optional specifier of an optional parameter.<br /></p><p>NOTE: This section should only contain parameters that are specified in the parent's resource URI template, and does not have to list every URI parameter.<br /></p>
