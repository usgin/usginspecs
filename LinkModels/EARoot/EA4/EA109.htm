Notes: An API's method MAY support or require non-standard HTTP headers. In the API definition, specify the non-standard HTTP header parameters by using the headers property.<br /></p><p><br /></p><p>The headers property is a map in which the key is the name of the header, and the value is itself a map specifying the header attributes, according to the Named Parameters section.<br /></p><p>If the header parameter displayname contains the placeholder token {*}, processing applications MUST allow requests to send any number of headers that conform to the format specified, with {*} replaced by 0 or more valid header characters, and offer a way for implementations to add an arbitrary number of such headers. This is particularly useful for APIs that allow HTTP headers that conform to custom naming conventions to send arbitrary, custom data.<br /></p>
