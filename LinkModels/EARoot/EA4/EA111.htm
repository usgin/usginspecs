If type is not specified, it defaults to string. Valid types are:<br /></p><p>Type Description<br /></p><p><ul><li>string Value MUST be a string.</li><br /></p><p></ul><ul><li>number Value MUST be a number. Indicate floating point numbers as defined by YAML.</li><br /></p><p></ul><ul><li>integer Value MUST be an integer. Floating point numbers are not allowed. As defined in [RFC2616], all date/time stamps are represented in Greenwich Mean Time (GMT), which for the purposes of HTTP is equal to UTC (Coordinated Universal Time). This is indicated by including "GMT" as the three-letter abbreviation for the timezone. Example: Sun, 06 Nov 1994 08:49:37 GMT.</li><br /></p><p><li>boolean Value MUST be either the string "true" or "false" (without the quotes).</li><br /></p><p></ul><ul><li>file (Applicable only to Form properties) Value is a file. Client generators SHOULD use this type to handle file uploads correctly.</li><br /></p><p></ul><br /></p>
